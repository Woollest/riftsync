# Changelog

RiftSync の仕様変更と実装内容を記録する。

## 2026-06-18

### Added

- OP.GG Champion Tier Listからロール別統計を取り込む `pnpm update:opgg` を追加
- OP.GGの個別シナジーページから `pairSynergies.csv` を一括生成する `pnpm update:opgg:synergies` を追加
- `pairSynergies.json` を別アセットとして読み込む `usePairSynergies` を追加
- 公開リポジトリ向けに開発者用の構成メモ `docs/developer/architecture.md` を追加
- おすすめカードに「OP.GG直接 / 推定補完」と信頼度ラベルを追加
- データ情報パネルに現在の採用方式を表示する項目を追加
- PWA向けのservice worker、スマホ用PNGアイコン、Appleホーム画面用メタ情報を追加
- push / pull request / 手動実行で検証するGitHub Actions CIを追加
- PWA manifest、アイコン、service workerを検証する `pnpm check:pwa` を追加
- 相性データ読み込み中のスケルトン表示を追加
- Data Dragonとロール分類を使って、CSV未登録チャンピオンも補完候補として推薦プールに含める仕組みを追加
- おすすめ3体の下に追加候補を最大5体表示する機能を追加
- 補完候補を判別できる「補完データ」ラベルとデータ情報パネルの補完候補数表示を追加

### Changed

- `roleStats.csv` をOP.GG Global / Gold+ / Ranked Solo/Duo / Patch 16.12.1 の259件へ更新し、サンプルデータ扱いを解除
- `pairSynergies.csv` をOP.GG個別シナジーページ由来の3108件へ更新し、1036個の味方/味方ロール/自分ロール組み合わせを直接相性データでカバー
- `pnpm update:opgg` はロール別統計とシナジーの両方を更新するよう変更
- GitHub Pagesデプロイ前チェックを `pnpm run verify` に統一
- 初期JSから大きなシナジーJSONを外し、production buildのメインJSを約833kBから約259kBへ削減
- 直接相性の理由文にOP.GG上位、ペア勝率、試合数を含めるよう変更
- READMEをWebアプリ利用者向けに書き換え、開発コマンドやクローン手順を削除
- READMEにおすすめカードのデータソースと信頼度の見方を追記
- READMEにスマホのホーム画面追加について追記
- `src/app`、`src/domain`、`src/config`、`src/data` に責務ごとにファイルを再配置
- 開発者向けドキュメントを `docs/developer/`、要件定義を `docs/product/` へ整理
- チャンピオン画像を遅延読み込みに変更し、検索入力は遅延反映と検索インデックスで再描画負荷を軽減
- Data Dragonと相性JSONの読み込みをキャッシュし、初期表示時の重複リクエストを削減
- PWAアイコンの外側余白、視覚中心、service workerキャッシュを調整し、スマホのホーム画面で黒い縁やズレが出にくい画像へ更新
- 要件定義書をv1.1に更新し、OP.GG取り込み、信頼度表示、PWA、パフォーマンス改善を現行仕様として反映
- 直接相性データが少ない場合は、広い候補プールからタグと戦闘プロフィールを使って多様性を考慮した推薦を選ぶよう変更
- 非推奨候補は補完データより実データ/直接相性データを優先して選ぶよう変更
- 相性CSVのおすすめ先が `roleStats.csv` 未登録でも、エラーではなく補完データ前提の警告として扱うよう変更

### Verified

- `pnpm share:check`

## 2026-06-17

### Added

- サンプルADC全8体からSupportを選ぶケースの直接相性データを追加
- 相性CSVが無い味方チャンピオンでも候補が固定化しにくいよう、チャンピオン性質ベースの推定相性を追加
- CSV由来ではない相性候補に「推定相性」ラベルを追加
- おすすめカードに総合スコアの内訳表示を追加
- おすすめカードに試合数表示と低サンプル時のデータペナルティ表示を追加
- 選択中の味方ロール、味方チャンピオン、自分ロールをURLに同期する機能を追加
- 現在の条件を開けるURLをコピーする「リンクをコピー」アクションを追加
- PC横長表示では入力欄を左、結果を右に分けるレスポンシブレイアウトを追加
- おすすめ結果をテキストとしてコピーできる共有アクションを追加
- GitHub Issuesへ送るフィードバック導線を追加
- 共有後の変更内容を追えるように `CHANGELOG.md` を追加

### Changed

- 直接相性データが3件以上ある場合、おすすめ3体はCSVの上から3件をそのまま優先表示するよう変更
- 総合スコアの重みを相性85%、勝率10%、メタ5%に変更し、味方チャンピオンとの相性をより強く反映
- PC横長表示で左のsticky入力パネルと下部UIが重ならないよう、選択中表示と非推奨候補の配置を調整
- `App.tsx` に集中していたUI部品、Data Dragon取得、共有URL、クリップボード処理を分割

### Verified

- `pnpm share:check`
