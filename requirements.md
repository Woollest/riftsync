# RiftSync 要件定義書

作成日: 2026-06-17  
版: v1.0

## 1. 概要

RiftSync は、League of Legends のソロQ通常ランクにおいて、味方1人のピックをもとに、自分のロールで相性の良いチャンピオンを提案する初心者向けWebアプリである。

チャンピオンセレクト中に短時間で使えることを重視し、入力は最小限に抑える。おすすめチャンピオンを3体表示し、あわせて非推奨候補も折りたたみ形式で確認できるようにする。

## 2. 目的

- 初心者が味方ピックとの相性を考慮して、自分のピックを選びやすくする
- メタ、勝率、コンボ相性を総合して、候補を3体に絞る
- 数値だけではなく、構成ベースの短い理由を表示する
- チャンピオンセレクト中でも迷わず使える、視認性の高いゲーム風UIを提供する

## 3. 想定ユーザー

- League of Legends 初心者
- ピック判断に慣れていないソロQプレイヤー
- 自分と友人が利用する範囲のユーザー

## 4. 対象範囲

### 4.1 対象

- League of Legends
- ソロQ
- 通常ランクのBAN/PICK
- Gold以上のデータ
- 地域指定なし
- チャンピオンセレクト中の利用

### 4.2 初期版で対象外

- 敵ピックの考慮
- BAN状況の考慮
- 味方2体以上の組み合わせ考慮
- ARAM
- Clash
- カスタム
- スクリム
- 観戦、試合後分析
- Riotアカウント連携
- チャンピオンセレクト画面の自動読み取り
- 自動データ更新
- 管理者用データ更新画面

## 5. アプリ名

正式名称: RiftSync

名前の意図:

- Rift: LoLの世界観を想起しやすい
- Sync: 味方との連携、相性、構成のかみ合いを表す
- 短く、ゲーム風UIにも合わせやすい

## 6. 利用フロー

1. ユーザーが味方のロールを選択する
2. ユーザーが味方チャンピオンを検索またはアイコン一覧から選択する
3. ユーザーが自分のロールを選択する
4. アプリが自分のロールで使えるおすすめチャンピオンを3体表示する
5. ユーザーが必要に応じて「非推奨候補を見る」を開く
6. アプリが非推奨チャンピオンを3体表示する

## 7. 入力要件

### 7.1 味方のロール

必須入力。

選択肢:

- Top
- Jungle
- Mid
- ADC
- Support

### 7.2 味方チャンピオン

必須入力。

選択方法:

- 検索バー
- チャンピオンアイコン一覧
- 選択中の味方ロールにマッチするチャンピオンを上部に表示
- それ以外のチャンピオンを下部に表示

検索条件:

- 日本語名で検索可能
- 英語名で検索可能
- 部分一致検索

表示条件:

- チャンピオン基本データに存在する全チャンピオンを表示候補に含める
- 選択した味方ロールにマッチするチャンピオンを上部に表示する
- ロール一致判定は、ロール別統計データだけでなく、ポジション用チャンピオンカタログも使う
- それ以外のチャンピオンを下部に表示する
- 味方チャンピオンの選択一覧には「オフメタ」ラベルを表示しない

### 7.3 自分のロール

必須入力。

選択肢:

- Top
- Jungle
- Mid
- ADC
- Support

制約:

- UI上は、自分のロール選択で味方ロールと同じロールを選べないようにする

## 8. 出力要件

### 8.1 おすすめチャンピオン

おすすめチャンピオンを3体表示する。

並び順:

- 総合スコア順

表示項目:

- チャンピオンアイコン
- 日本語名
- 英語名
- ロール
- 総合評価
- コンボ相性
- 勝率
- メタ評価
- 難易度
- 初心者におすすめラベル
- オフメタラベル
- データ少ラベル
- 1行のおすすめ理由

### 8.2 非推奨チャンピオン

非推奨チャンピオンを3体表示する。

表示方法:

- 初期表示では閉じておく
- 「非推奨候補を見る」ボタンで開く

選定基準:

- 勝率が低いチャンピオンを優先して表示する

表示項目:

- チャンピオンアイコン
- 日本語名
- 英語名
- 勝率
- コンボ相性
- 非推奨理由

## 9. 表示ルール

### 9.1 総合評価

表示形式:

- 5つ星
- 半星あり

例:

- ★★★★★
- ★★★★☆
- ★★★★½

### 9.2 コンボ相性

表示形式:

- パーセント数値

例:

- コンボ相性 87%

### 9.3 勝率

表示形式:

- 数値をそのまま表示

例:

- 勝率 52.1%

### 9.4 メタ評価

表示形式:

- 炎マーク5段階
- 数字は併記しない

例:

- 🔥🔥🔥🔥

### 9.5 難易度

表示形式:

- 簡単
- 普通
- 難しい

データ元:

- Riot公式のチャンピオン難易度を利用する

変換基準:

- 1〜3: 簡単
- 4〜7: 普通
- 8〜10: 難しい

### 9.6 初心者におすすめラベル

表示条件:

- 難易度が「簡単」のチャンピオンに表示する

### 9.7 オフメタラベル

表示条件:

- 対象ロールにデータは存在するが、使用率が低い場合に表示する
- 初期基準は対象ロール内の使用率1.0%未満とし、実データを見て調整可能とする

### 9.8 データ少ラベル

表示条件:

- サンプル数が500試合未満の場合に表示する

除外条件ではない:

- データ少ラベルが付いていても、候補からは除外しない

## 10. スコア計算

### 10.1 総合スコア

総合スコアは以下の重みで算出する。

```text
総合スコア = コンボ相性 50% + 勝率評価 30% + メタ評価 20%
```

重み:

- コンボ相性: 50%
- 勝率評価: 30%
- メタ評価: 20%

### 10.2 スコアの扱い

- 内部スコアは0から100で扱う
- コンボ相性は0から100のパーセントとして扱う
- 勝率は表示用の実勝率とは別に、同ロール内で正規化した勝率評価スコアへ変換する
- メタ評価は0から100の内部スコアとして扱い、表示時は炎マーク5段階に変換する

### 10.3 星評価への変換

総合スコアを5つ星、半星ありに変換する。

初期案:

```text
95-100 = ★★★★★
85-94  = ★★★★½
75-84  = ★★★★☆
65-74  = ★★★½☆
55-64  = ★★★☆☆
45-54  = ★★½☆☆
35-44  = ★★☆☆☆
25-34  = ★½☆☆☆
0-24   = ★☆☆☆☆
```

### 10.4 勝率評価スコアの正規化

勝率評価スコアは、同じ自分ロールの候補チャンピオン内で相対評価する。

初期案:

```text
勝率評価スコア = (対象勝率 - 同ロール最低勝率) / (同ロール最高勝率 - 同ロール最低勝率) * 100
```

補足:

- 表示上の勝率は実勝率をそのまま表示する
- 総合スコア計算には正規化後の勝率評価スコアを使う
- 最高勝率と最低勝率が同じ場合は、勝率評価スコアを50として扱う

## 11. おすすめ理由

### 11.1 方針

- 1行程度で表示する
- 数値ベースより構成ベースを重視する
- 初期版では固定テンプレート式にする
- テンプレートは10種類用意する

### 11.2 理由テンプレート案

1. 味方のエンゲージに合わせて火力を出しやすい
2. 味方のCCに合わせてキルを狙いやすい
3. 前衛がいるため安全にダメージを出しやすい
4. 構成のダメージバランスを整えやすい
5. 味方のピールを受けながら継続火力を出しやすい
6. 味方の集団戦開始に合わせて範囲ダメージを出しやすい
7. 味方の序盤圧力に合わせてレーン主導権を取りやすい
8. 味方のスケーリングを支えやすい
9. 味方のロームやガンクに合わせて有利を広げやすい
10. 味方が作った隙にバーストで倒し切りやすい

### 11.3 非推奨理由テンプレート案

- 味方との勝率が低く、構成としてかみ合いにくい
- 味方の強みと噛み合うタイミングが少ない
- 集団戦で役割が重なりやすい
- ダメージや前衛のバランスが崩れやすい
- 味方の動きに合わせにくく、初心者には扱いづらい

## 12. UI要件

### 12.1 全体方針

- 日本語UI
- スマホ風の縦並びレイアウト
- 暗めでネオン感のあるゲームHUD風UI
- 配色は青・シアン系を中心にする
- アイコンを多用する
- チャンピオンセレクト中に素早く使える
- 操作数を少なくする

### 12.2 画面構成

1. ヘッダー
2. 味方ロール選択
3. 味方チャンピオン検索
4. チャンピオンアイコン一覧
5. 自分のロール選択
6. おすすめチャンピオン3体
7. 非推奨候補を開くボタン
8. 非推奨チャンピオン3体

### 12.3 レイアウト

- 基本は縦並び
- スマホ対応を前提にする
- PCでも中央寄せのスマホライクな表示にする
- チャンピオンアイコン一覧は小さいアイコンのグリッド表示にする
- 味方チャンピオン一覧では、ポジション用カタログに一致する候補を上段にまとめる

### 12.4 おすすめカード

カードに含める情報:

- チャンピオンアイコン
- 日本語名 / 英語名
- 総合評価
- コンボ相性
- 勝率
- メタ評価
- 難易度
- 各種ラベル
- おすすめ理由

## 13. データ要件

### 13.1 データ参照元

候補:

- U.GG
- OP.GG
- Riot Data Dragon

用途:

- U.GG / OP.GG: 勝率、メタ、相性データの参考
- Riot Data Dragon: チャンピオン名、アイコン、公式難易度などの静的データ

注意:

- 外部サイトのデータ取得方法は利用規約や技術制約を確認する
- 初期版ではU.GG / OP.GGを参照し、手動でCSVまたはJSONに転記する

### 13.2 更新頻度

- 毎パッチ更新
- 自動更新は初期版では不要
- 管理者用画面は初期版では不要
- CSVまたはJSONを直接編集して更新する
- U.GG / OP.GGを見ながら手動でデータを更新する

### 13.3 チャンピオン基本データ

```text
champion_id
name_ja
name_en
icon_url
difficulty
```

### 13.4 ロール別データ

```text
champion_id
role
win_rate
win_rate_score
pick_rate
tier
meta_score
is_off_meta
sample_size
patch
rank_range
```

### 13.5 味方相性データ

```text
ally_champion_id
ally_role
recommended_champion_id
recommended_role
combo_score
pair_win_rate
sample_size
reason_type
patch
rank_range
```

### 13.6 理由テンプレートデータ

```text
reason_type
reason_text_ja
```

### 13.7 初期データ作成方針

- コンボ相性データはU.GG / OP.GGを参照して手動でCSV化する
- ロール別勝率、使用率、メタ評価もU.GG / OP.GGを参考にする
- チャンピオンアイコンと公式難易度はRiot Data Dragonを利用する
- 初期版では自動スクレイピングや外部API連携は行わない

## 18. 受け入れ条件

- ユーザーが味方ロールを選択できる
- ユーザーが味方チャンピオンを検索、選択できる
- 味方チャンピオン一覧は、味方ロールにマッチした候補を上部、それ以外を下部に表示する
- 味方チャンピオン一覧にはオフメタラベルを表示しない
- ユーザーが自分のロールを選択できる
- 味方ロールと同じ自分のロールは選べない
- 日本語名と英語名の両方で検索できる
- 味方チャンピオン選択後、おすすめチャンピオンが3体表示される
- おすすめ結果をテキストとしてクリップボードへコピーできる
- フィードバック導線からGitHub Issuesを開ける
- おすすめチャンピオンが総合スコア順に並ぶ
- 各おすすめに総合評価、コンボ相性、勝率、メタ評価、難易度、理由が表示される
- 難易度が簡単のチャンピオンには初心者おすすめラベルが表示される
- サンプル数500未満の場合はデータ少ラベルが表示される
- おすすめカード内のオフメタ候補にはオフメタラベルが表示される
- 非推奨候補は初期表示されず、ボタンで開ける
- 非推奨候補が3体表示される
- スマホ幅でも画面が破綻しない
- 青・シアン系の暗めネオンHUD風UIになっている
- GitHub Pagesで公開できる
